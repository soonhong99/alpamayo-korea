## 0. 왜 이 시뮬레이션이 필요한가

ADE/FDE는 "숫자가 얼마나 다른가"를 측정한다.  
이 시뮬레이션은 "그 차이가 실제로 사고를 내는가"를 측정한다.

```
기존 평가:                     이 시뮬레이션:
ADE = 0.907m → FAIL?          앞차와 간격이 0m 미만? → 충돌
(기준 자체가 임의적)            (물리적으로 명확한 기준)
```

## 2. 앞차 키네마틱 추출 방법

### 왜 이 방법인가

다른 차량의 GPS/위치 정보가 데이터셋에 없다.  
카메라 이미지와 알려진 물리량(차량 크기, FOV)만으로 수학적으로 계산한다.  
모든 단계가 측정값 기반이며 "가정한 속도"를 쓰지 않는다.

### Step 1: Telephoto 이미지에서 앞차 탐지

```
대상 이미지: image_frames[3, :, :, :, :]
            (camera_index=6인 telephoto, 4프레임)

탐지 방법:
  1. HSV 색공간 변환
  2. 흰색 범위 마스크: H∈[0,180], S∈[0,30], V∈[200,255]
  3. 관심 영역(ROI) 적용:
     - 가로: 이미지 중앙 ±20% (우리 차선 영역)
     - 세로: 이미지 상단 30%~60% (근/중거리 영역)
  4. 가장 큰 contour의 bounding box 추출
  5. box 너비 W_px [픽셀] 측정
```

### Step 2: Known-size Depth 계산

```
물리적 사실 (가정이 아닌 표준 차량 규격):
  실제 세단 차폭: W_real = 1.8m (한국 표준 세단)

공식:
  D = f_x × W_real / W_px
  D = 3,582 × 1.8 / W_px   [m]

4프레임 각각에 적용:
  D[0] = 3582 × 1.8 / W_px[0]   (t=0.0s)
  D[1] = 3582 × 1.8 / W_px[1]   (t=0.1s)
  D[2] = 3582 × 1.8 / W_px[2]   (t=0.2s)
  D[3] = 3582 × 1.8 / W_px[3]   (t=0.3s)
```

### Step 3: 상대 속도·가속도 계산

```
상대 속도 (가장 최근 2프레임):
  v_rel = (D[3] - D[2]) / 0.1s   [m/s]
  양수 = 앞차가 멀어짐 (앞차가 더 빠름)
  음수 = 앞차가 가까워짐 (앞차가 더 느림 = 위험)

상대 가속도 (3프레임으로 추정):
  a_rel = (D[3] - 2×D[2] + D[1]) / (0.1)²   [m/s²]

앞차 절대 속도:
  v_lead = v_ego + v_rel   [m/s]

→ 모두 측정값. 가정 없음.
```

### Step 4: 앞차 64-step 위치 시뮬레이션

```
D(t) = D_0 + v_rel × t + ½ × a_rel × t²

단, a_rel은 0.3s 관측 기반이므로 장기 외삽 오차 있음.
→ 시뮬레이션 결과 해석 시 "관측된 가속도가 유지된다는 가정 하에"임을 명시.
```

## 3. BEV 시뮬레이션 구성

### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│                    t = 2.30s                             │
│                                                          │
│   ┌─────────────────────────────────────────────────┐   │
│   │  차선 경계 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │   │
│   │                                                 │   │
│   │         [  lead vehicle  ]  ←──흰색 세단        │   │
│   │                                                 │   │
│   │  간격표시 ←─── 11.2m ───→ ←── 14.5m ──→        │   │
│   │                                                 │   │
│   │    [GT ego 🟢]  [N=16 ego ⚫]  [N=0 ego 🔴]     │   │
│   │                                                 │   │
│   │  차선 경계 ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │   │
│   └─────────────────────────────────────────────────┘   │
│                                                          │
│   [Gap 차트]  GT ─── N=16 ─── N=0 ───                   │
│               안전구간(>3m) ─ ─ ─ ─ ─                    │
│               ⚠ N=16: 충돌 위험 구간                      │
└──────────────────────────────────────────────────────────┘
```

### 시각적 요소

| 요소 | 표현 방식 | 크기/단위 |
|------|----------|----------|
| Ego 차량 (GT) | 초록색 직사각형 | 4.5m × 2.0m (실제 비율) |
| Ego 차량 (N=16) | 검정 직사각형 | 동일 |
| Ego 차량 (N=0) | 빨간색 직사각형 | 동일 |
| Lead 차량 | 흰색 테두리 직사각형 | 4.5m × 2.0m |
| 차선 경계 | 파란 점선 | ±1.75m (표준 차로폭 3.5m) |
| 충돌 구간 | 빨간 배경 flash | gap < 0m |
| 위험 구간 | 노란 배경 | gap < 3m |
| 간격 숫자 | 실시간 텍스트 | 소수점 1자리 [m] |

### 충돌 판정 로직

```python
ego_front(t) = x_ego(t) + L_ego / 2      # ego 앞범퍼 위치
lead_rear(t) = D(t) - L_lead / 2         # 앞차 뒷범퍼 위치

gap(t) = lead_rear(t) - ego_front(t)

gap(t) < 0:  COLLISION  → 빨간 화면 + "충돌!" 텍스트
gap(t) < 3:  CRITICAL   → 노란 화면 + "위험" 텍스트
gap(t) < 6:  WARNING    → 주황 텍스트
gap(t) >= 6: SAFE       → 초록 텍스트
```

```
(d) 위험 구간 요약 표
    각 조건별 최소 gap, 충돌 여부, 위험 구간 시작 시각
```

## 5. 6.4초 이후에 대해

### 시뮬레이션이 6.4초를 넘어가지 않는 이유

```
모델 예측 범위: 0 ~ 6.4s (64 waypoints)
GT 범위:        0 ~ 6.4s (ego_future_xyz)
앞차 추정 범위: 0 ~ 6.4s (0.3s 관측 기반 외삽)

6.4초 이후:
  - 모델 경로 없음 (예측 불가)
  - 추가 가정 필요 (상수 속도? 감속? 정지?)
  → 측정값 기반이 아니게 됨 → 포함하지 않음
```

### 6.4초 창으로 충분한 이유

```
현재 ego 속도: 8.66 m/s = 31.2 km/h
앞차 예상 거리: ~15~20m (telephoto 시각적 추정)

최악 시나리오 (앞차 급정차):
  앞차 제동거리 @ 31km/h: ~13m (마른 노면, μ=0.8)
  → 앞차가 6.4s 내에 이미 정차 가능
  → 6.4s 창이 충돌 발생 여부 판단에 충분
```

## 6. 스크립트 구조

```
scripts/collision_bev_sim.py
│
├── load_data()
│     └── load_physical_aiavdataset + waypoints_0.npy + waypoints_16.npy
│
├── compute_ego_velocity()
│     └── ego_history_xyz[-2:] → v_ego
│
├── detect_lead_vehicle(telephoto_frames)
│     └── HSV masking → ROI → contour → bbox per frame
│
├── compute_lead_kinematics(bboxes)
│     └── D per frame → v_rel, a_rel → D(t) for 64 steps

│
├── render_bev_animation()
│     └── matplotlib FuncAnimation → MP4
│
└── render_summary_figure()
      └── 4-panel static PNG
```

## 7. 한계 및 오차 요인

| 요인 | 영향 | 크기 추정 |
|------|------|----------|
| 차폭 가정 (1.8m) | 실제 차폭이 다를 경우 D 오차 | ±0.1m → D 오차 ±(D×6%) |
| FOV 가정 (30° 정확히) | f_x 오차 → D 오차 | 카메라 스펙 확인 필요 |
| 가속도 외삽 (0.3s → 6.4s) | 장기 거동 불확실 | 시나리오 sweep으로 보완 |
| 횡방향 무시 | 앞차가 차선 변경 시 누락 | 현 장면에서 직진으로 보임 |

> 핵심 질문 "3.3m 차이가 충돌을 내는가"에 대한 답은  
> 앞차의 종방향 거동이 결정한다. 횡방향은 현 장면에서 부차적이다.
