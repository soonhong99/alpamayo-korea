## 1. 미팅 핵심 결론

### 이전 방향 (폐기)
- Alpamayo 1.5 모델 자체를 수정 (Korean fine-tuning, RL post-training)
- Decode 단계를 건너뛰는 방식으로 속도 향상 시도
- 모델의 reasoning trace를 수정

**교수님 피드백**: "모델을 건드리는 것은 우리 할 일이 아니다. Decode 단계 생략은 모델이 향상되면 언제든 달라질 수 있어 연구 가치가 없다."

### 새 방향 (확정)
> **모델은 그대로 둔다. 시스템 레벨에서 pipelining과 최적화 기법을 적용하여 throughput을 극대화한다.**

구체적으로:
- Async pipeline (RT-Swap, Demand Layering 논문의 핵심 기법)을 Alpamayo에 적용
- iGPU(통합 메모리 아키텍처)에서 검증된 기법을 Thor에 이식
- **GPU L2 캐시 + 시스템 캐시 + DRAM 계층**을 활용한 지능적 메모리 관리

## 3. 교수님이 제기한 기술적 질문들 (연구 과제)

### Q1. 96 Tensor Cores vs 20 SMs — 어떻게 나눠 쓰는가?
Thor 아키텍처 다이어그램: "2560-core Blackwell GPU, **96 Tensor Cores**"  
하지만 실측: `props.multi_processor_count = 20` (SM 20개)

→ **96 / 20 = 4.8**: 정수가 아님 → Blackwell의 SM 내부 구조 이해 필요  
→ 자세한 분석: [260524_02_Thor_아키텍처_심층분석.md](260524_02_Thor_아키텍처_심층분석.md)

### Q2. 16 MB System Cache — GPU/CPU 공유 캐시의 개념은?
다이어그램에서 CPU L1/L2/L3와 GPU 32MB L2와는 별도의 "16 MB System Cache"가 존재.

→ **SoC 레벨 LLC**: CPU, GPU, ISP 등 모든 처리 유닛이 공유  
→ DRAM 앞단에서 모든 마스터의 메모리 접근을 캐싱  
→ 자세한 분석: [260524_02_Thor_아키텍처_심층분석.md](260524_02_Thor_아키텍처_심층분석.md)

### Q3. CPU + GPU 동시 실행 시 DRAM 대역폭?
GPU 단독: ~231 GB/s (실측)  
CPU 단독 (1 core): ~33 GB/s (실측)  
**동시 실행 시**: 대역폭 경쟁 발생? → 미측정 → 실험 과제

→ 실험 계획: [260524_05_실험_계획_및_로드맵.md](260524_05_실험_계획_및_로드맵.md)

### Q4. CPU 멀티코어 대역폭?
14 core Neoverse V3AE 전체 활용 시 대역폭?  
단일 코어: ~33 GB/s → 14코어 이론: ~462 GB/s? (실제는 DRAM 제한)  
→ 미측정 → 실험 과제

## 4. 새 연구 가설

```
[현재 Alpamayo 추론]
Step 1:  [Load Layer 0 from DRAM] → [Compute Layer 0] → [Load Layer 1] → [Compute Layer 1] → ...
         ◄─────── 순차 실행 ──────────────────────────────────────────────────────────────────►
         총 시간 = Σ(load_time_i + compute_time_i)   ≈ 5.8s DRAM bound

[Async Pipeline 적용 후]
Stream A (DMA):  [Load L0]──────[Load L1]──────[Load L2]──────...
Stream B (GPU):          [Compute L0][Compute L1][Compute L2]...
                 ◄─── 중첩 실행 ──────────────────────────────────────►
         총 시간 = max(total_load, total_compute) < 순차 실행
```

이것이 가능하면:
- 현재 5.8초 → 이론 최솟값 max(5.8s, compute_only_time)으로 감소
- throughput: tokens/sec 향상
